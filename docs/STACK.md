# Stack & Tools

## Objective
Build a portfolio that works as a real product,
demonstrating mastery of design, frontend, and backend,
with a focus on visible backend engineering.

---

## Frontend
- HTML5
- CSS
- HTML
- Vanilla JavaScript
- Native dark mode

### Reasons
- Full control of UI and UX
- Less complexity than SPA
- Real interactions with backend
- Performance and simplicity

---

## Backend
- Python
- Flask
- Flask Blueprints
- Service Layer
- Repository Pattern

### Mandatory features
- Authentication (admin)
- Public REST API
- Private API (admin)
- Data validation
- Basic rate limiting
- Action logs

---

## Database
- PostgreSQL
- Alembic (migrations)

### Minimum tables
- projects
- blog_posts
- messages
- visits
- users (admin)

## Infrastructure

- Environment variables
- Git + GitHub
